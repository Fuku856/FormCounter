# 動作確認チェックリスト

自動テストで確認できることと、実際のブラウザで目視するしかないことを分けてある。

## 自動テスト（コマンド 1 つ）

```sh
node tests/run.js
node tools/check-manifest.js
```

`tests/unit.html` をブラウザで開いても同じケースが走る。

カバーしているもの:

- 文字数カウント — 全角スペース・タブ・NBSP・改行・追加面の漢字・絵文字、設定 4 通り
- 配置計算 — PC / スマホ縦 / **スマホ横＋キーボード** / 端への張りつき / 総当たりで画面内に収まること
- 寸法計算 — 実装計画の表と一致すること
- manifest — 参照先ファイルと翻訳の欠落

## パソコンでの目視確認

1. `chrome://extensions` → デベロッパー モードをオン
2. 「パッケージ化されていない拡張機能を読み込む」でこのフォルダを選ぶ
3. Google フォームを作り、**短答 / 段落 / ラジオ（その他つき） / 日付 / セクション区切り**を入れる
4. 回答画面（`/viewform`）を開く

| # | 確認すること | 期待 |
| --- | --- | --- |
| 1 | 短答欄をクリック | 欄の**右外側**に `0` が出る |
| 2 | 文字を入力 | 数字がリアルタイムに増える |
| 3 | 段落欄をクリック | 欄の右外側・**下端そろえ**で出る |
| 4 | 段落欄で改行を増やす | 欄が伸びてもバッジが下端に追随する |
| 5 | 別の欄へフォーカス移動 | 前の欄のバッジが消え、新しい欄にだけ出る |
| 6 | **ラジオの「その他」欄**をクリック | **バッジが出ない** |
| 7 | 日付欄をクリック | バッジが出ない |
| 8 | ページをスクロール | バッジが欄に追随する |
| 9 | ウィンドウ幅を狭める | 欄の右に余白が無くなると**欄の真上**へ移動する |
| 10 | 「次へ」でセクション移動 | 移動先の欄でも動く |
| 11 | 拡張機能アイコンをクリック | 設定画面が開き、両方オフになっている |
| 12 | 「空白を数える」をオン | **フォームのタブを再読み込みせずに**数字が増える |
| 13 | 半角/全角スペースを入力 | オンのときだけ数に入る |
| 14 | 改行の設定も同様に確認 | |
| 15 | フォームのテーマを濃い色に変更 | 数字が読める |
| 16 | OS をダークモードに切り替え | 数字が読める／設定画面も反転する |

### 文字数の妥当性

`こんにちは 世界` を入力したとき:

| 設定 | 期待 |
| --- | --- |
| 既定（どちらもオフ） | `7` |
| 空白オン | `8` |

## スマートフォンでの目視確認

拡張機能に対応した Android の Chromium 系ブラウザ（Cromite など）へ導入する。

> **注意:** Cromite は 3 点メニューから拡張機能を扱えるが、開発元は
> 「サポート対象外」の位置づけにしており、クラッシュの報告もある。
> うまくいかない場合は、まずパソコンでの確認を済ませてから切り分けること。
> 開発中の未パッケージ拡張をスマートフォンへ読み込む手順は未調査。

| # | 確認すること | 期待 |
| --- | --- | --- |
| 1 | 段落欄をタップしてキーボードを出す | **バッジがキーボードに隠れない** |
| 2 | そのまま入力 | 数字が動き、入力文字に重ならない |
| 3 | 画面を横向きにする | バッジが可視領域内に残る |
| 4 | 横向きのままキーボードを出す | 可視領域が細い帯になっても**右上に退避して見える** |
| 5 | 「次へ」「送信」ボタンにバッジが重なる状況を作る | **ボタンがタップできる**（バッジは `pointer-events: none`） |
| 6 | 日本語変換（未確定）中 | 数字が動く（仕様） |
| 7 | 拡張機能アイコンをタップ | 設定画面が**横スクロールなし**で表示される |
| 8 | 設定画面を横向きにする | 2 カラムになり、スクロールせず全項目が見える |

**パソコンでは再現できない項目:** 1〜6。DevTools のデバイス
エミュレーションはソフトウェアキーボードを模倣しないため、
`visualViewport` が縮む状況を作れない。座標計算そのものは
`tests/cases-position.js` で検証済み。

## PLAN.md「完成の基準」との対応

| 基準 | 状態 |
| --- | --- |
| Chrome 拡張として Forms の自由記述欄で動作 | 実装済み / 要目視 |
| パソコン版 Chromium で基本動作 | 要目視 |
| Android Chromium で基本動作 | 要実機 |
| フォーカス中の欄だけにリアルタイム表示 | 実装済み・フィクスチャで確認 |
| 表示は数字だけ | 実装済み |
| 半透明背景・ライト／ダーク両方で読める | 実装済み（白背景 5.17:1 / 黒背景 18.9:1） |
| 空白を数えるか設定できる | 実装済み・テスト済み |
| 改行を数えるか設定できる | 実装済み・テスト済み |
| 初期設定では数えない | 実装済み・テスト済み |
| 画面サイズ・向きで表示の位置と大きさが調整 | 実装済み・テスト済み / 実機確認は残 |
| 画面サイズ・向きで設定画面が調整 | 実装済み・プレビューで確認 |
| キーボードや主要操作を妨げない | `pointer-events: none` で保証 / 実機確認は残 |
| 入力内容を保存・収集・外部送信しない | 実装済み（権限は `storage` のみ・通信コードなし） |
